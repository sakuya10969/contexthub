# プロダクト概要 — RAG ドキュメント検索システム

## ゴール

文書をアップロードし、S3 に保存し、テキスト抽出・チャンク分割・Embedding 生成・OpenSearch へのインデックス登録を行い、
その文書群に対して自然言語検索およびチャット形式の質問応答ができる RAG システムを構築する。
回答時には引用元を確認できるようにし、検索根拠が分かる UI を持たせる。

## 対象ユーザー

- 社内ナレッジを検索・活用したいチームメンバー
- 文書管理と質問応答を一元化したい組織

## 対象ファイル形式

- PDF
- TXT
- Markdown

## 実現する機能

| # | 機能 | 概要 |
|---|------|------|
| 1 | 文書アップロード | ファイルを選択して S3 に保存 |
| 2 | 文書一覧表示 | 登録済み文書の一覧・状態確認 |
| 3 | テキスト抽出 | PDF / TXT / Markdown からテキストを抽出 |
| 4 | チャンク分割 | 抽出テキストを検索単位に分割 |
| 5 | Embedding 生成 | AWS Bedrock でベクトル化 |
| 6 | OpenSearch インデックス登録 | チャンク + Embedding を登録 |
| 7 | 文書検索 | 自然言語によるセマンティック検索 |
| 8 | チャット質問応答 | 関連チャンクを取得し Bedrock で回答生成 |
| 9 | 引用元表示 | 回答に紐づく文書名・抜粋・ページ情報を表示 |
| 10 | 文書削除 | 文書と関連チャンクを削除 |
| 11 | 再インデックス | 既存文書を再処理してインデックス更新 |

## 文書状態

| 状態 | 説明 |
|------|------|
| `uploaded` | S3 にアップロード済み、未処理 |
| `processing` | テキスト抽出〜インデックス登録中 |
| `indexed` | インデックス登録完了 |
| `failed` | 処理中にエラー発生 |

## 画面構成

### 1. Document Upload
- ファイル選択・アップロード
- アップロード結果の確認

### 2. Document List
- 登録済み文書の一覧表示
- 文書状態の表示
- 削除・再インデックスの実行

### 3. Search / Chat
- 自然言語検索
- チャット形式の質問応答
- 回答本文の表示
- 引用元（文書名・抜粋・ページ情報）の表示

## ユースケース

1. ユーザーが文書をアップロードする
2. 文書は S3 に保存される
3. バックエンドが文書を処理し、OpenSearch にインデックス登録する
4. ユーザーは自然言語で検索できる
5. ユーザーはチャット形式で質問できる
6. バックエンドは関連チャンクを取得し、Bedrock に問い合わせて回答を生成する
7. UI は回答本文とあわせて参照文書や抜粋を表示する
8. ユーザーは文書一覧を確認できる
9. ユーザーは文書を削除できる
10. ユーザーは文書を再インデックスできる

## 実装優先順位

### Phase 1 — 基盤構築
- フロントエンド・バックエンドの雛形作成
- OpenSearch のローカル Docker 起動
- S3 アップロード機能
- 文書一覧の基本機能

### Phase 2 — 文書処理パイプライン
- PDF / TXT / Markdown のテキスト抽出
- チャンク分割
- Embedding 生成（Bedrock）
- OpenSearch への登録

### Phase 3 — 検索・チャット
- 検索 API
- チャット API
- 回答 + 引用元表示 UI

### Phase 4 — 管理・改善
- 文書削除
- 再インデックス
- エラーハンドリング改善
